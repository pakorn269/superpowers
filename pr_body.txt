## What this PR does / why we need it:
**💡 What:** Implemented global caching of the fully constructed fallback and default HTML response in `server.cjs`. The server now calculates the injected response with the target helpers on startup or file update via `fs.watch`, rather than performing the string generation and filesystem I/O inside the `GET /` handler itself.

**🎯 Why:** The `GET /` handler is the primary route accessed heavily by clients. Executing `fs.promises.readFile` and performing large multi-megabyte HTML string combinations and slices on every individual HTTP request creates a significant CPU and I/O bottleneck under load, causing massive event-loop blocking overhead per request.

**📊 Impact:** Eliminates the filesystem read overhead and massive memory allocation entirely for all `GET /` requests, dropping the handler latency significantly and unblocking the event loop for faster concurrent responses.

**🔬 Measurement:** Verified by passing the entire `brainstorm-server` HTTP handling test suite to ensure the cache still serves exactly the same content, handles initialization windows gracefully, and maintains stability.
